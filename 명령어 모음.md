# 명령어 모음

## 1. 문서 목적

### 용도
- 과제 수행 중 사용한 명령어를 한곳에 정리
- 시험 전 빠르게 복습할 수 있도록 목적·실행·예시·확인 포인트 중심으로 정리
- PowerShell, Git, Docker, WSL/Linux 권한 흐름을 한 번에 다시 볼 수 있도록 정리

### 읽는 방법
- `목적` : 왜 이 명령을 쓰는지
- `기본형` : 가장 단순한 형태
- `예시` : 실제 과제에 맞춘 형태
- `확인 포인트` : 실행 후 무엇을 보면 되는지
- `정리` : 실습 후 지워도 되는 항목

---

## 2. PowerShell 기본 명령어 세트

### 2-1. 현재 위치 확인

#### 목적
- 현재 내가 어느 폴더에 있는지 확인

#### 기본형
```powershell
Get-Location
```

#### 예시
```powershell
Set-Location 'D:\코디세이\ai-sw-workstation'
Get-Location
```

#### 확인 포인트
- 출력 경로가 `D:\코디세이\ai-sw-workstation`인지 확인

---

### 2-2. 파일/폴더 목록 보기

#### 목적
- 현재 폴더 안의 파일과 폴더 확인
- 숨김 파일이나 하위 폴더까지 같이 확인 가능

#### 기본형
```powershell
Get-ChildItem
Get-ChildItem -Force
Get-ChildItem -Recurse -Force
tree /F
```

#### 예시
```powershell
Set-Location 'D:\코디세이\ai-sw-workstation'
Get-ChildItem -Force
tree /F
```

#### 확인 포인트
- `README.md`, `명령어 모음.md`, `Dockerfile`, `app`, `logs`, `screenshots` 등이 보이는지 확인

---

### 2-3. 폴더 만들기

#### 목적
- 작업용 폴더 생성

#### 기본형
```powershell
New-Item -ItemType Directory -Path '.\sample_dir'
```

#### 예시
```powershell
New-Item -ItemType Directory -Force -Path '.\app','.\logs','.\screenshots'
New-Item -ItemType Directory -Force -Path '.\cli_lab','.\perm_lab'
```

#### 확인 포인트
- `Get-ChildItem`로 새 폴더가 생겼는지 확인

---

### 2-4. 파일 만들기

#### 목적
- 빈 파일 생성

#### 기본형
```powershell
New-Item -ItemType File -Path '.\note.txt'
```

#### 예시
```powershell
New-Item -ItemType File -Force -Path '.\README.md','.\logs\terminal.md','.\logs\docker.md'
New-Item -ItemType File -Force -Path '.\cli_lab\note.txt'
```

#### 확인 포인트
- 파일이 실제 생성됐는지 확인

---

### 2-5. 파일 내용 쓰기

#### 목적
- 텍스트 파일에 내용 저장

#### 기본형
```powershell
'hello' | Set-Content '.\note.txt'
```

#### 예시
```powershell
'sample line' | Set-Content '.\cli_lab\note.txt'
Get-Content '.\cli_lab\note.txt'
```

#### 확인 포인트
- `Get-Content` 했을 때 저장한 텍스트가 그대로 보이는지 확인

---

### 2-6. 파일 내용 추가하기

#### 목적
- 기존 파일 뒤에 내용을 이어서 추가

#### 기본형
```powershell
'second line' | Add-Content '.\note.txt'
```

#### 예시
```powershell
'first line' | Set-Content '.\note.txt'
'second line' | Add-Content '.\note.txt'
Get-Content '.\note.txt'
```

#### 확인 포인트
- 첫 줄과 둘째 줄이 함께 보이는지 확인

---

### 2-7. 파일 읽기

#### 목적
- 텍스트 파일 내용을 확인

#### 기본형
```powershell
Get-Content '.\note.txt'
```

#### 예시
```powershell
Get-Content '.\README.md'
Get-Content '.\Dockerfile'
Get-Content '.\app\index.html'
```

#### 확인 포인트
- 파일 내용이 정상적으로 출력되는지 확인

---

### 2-8. 파일 복사

#### 목적
- 원본 파일을 유지한 채 복사본 만들기

#### 기본형
```powershell
Copy-Item '.\note.txt' '.\backup_note.txt'
```

#### 예시
```powershell
Copy-Item '.\cli_lab\note.txt' '.\cli_lab\dir_a\note-copy.txt'
```

#### 확인 포인트
- 원본과 복사본이 둘 다 존재하는지 확인

---

### 2-9. 이름 변경

#### 목적
- 파일 또는 폴더 이름 바꾸기

#### 기본형
```powershell
Rename-Item '.\backup_note.txt' 'renamed_note.txt'
```

#### 예시
```powershell
Rename-Item '.\cli_lab\dir_a\note-copy.txt' 'note-renamed.txt'
```

#### 확인 포인트
- 이전 이름은 사라지고 새 이름으로 바뀌었는지 확인

---

### 2-10. 이동

#### 목적
- 파일 또는 폴더를 다른 위치로 옮기기

#### 기본형
```powershell
Move-Item '.\renamed_note.txt' '.\dir_a\'
```

#### 예시
```powershell
Move-Item '.\cli_lab\dir_a\note-renamed.txt' '.\cli_lab\dir_b\'
```

#### 확인 포인트
- 원래 위치에서는 사라지고 대상 위치에 보이는지 확인

---

### 2-11. 삭제

#### 목적
- 불필요한 파일/폴더 지우기

#### 기본형
```powershell
Remove-Item '.\a.txt'
Remove-Item '.\temp_dir' -Recurse -Force
```

#### 예시
```powershell
Remove-Item '.\cli_lab\empty.txt'
Remove-Item '.\temp_test_dir' -Recurse -Force
```

#### 확인 포인트
- `Get-ChildItem` 했을 때 삭제 대상이 더 이상 보이지 않는지 확인

---

### 2-12. PowerShell 과제용 실전 세트

#### 목적
- 파일/폴더 생성, 읽기, 복사, 이동, 삭제를 한 번에 실습

#### 실행
```powershell
Set-Location 'D:\코디세이\ai-sw-workstation'

New-Item -ItemType Directory -Force -Path '.\cli_lab' | Out-Null
Set-Location '.\cli_lab'

Get-Location
Get-ChildItem -Force

New-Item -ItemType Directory -Force -Path '.\dir_a','.\dir_b' | Out-Null
New-Item -ItemType File -Force -Path '.\note.txt','.\empty.txt' | Out-Null

'sample line' | Set-Content '.\note.txt'
Get-Content '.\note.txt'

Copy-Item '.\note.txt' '.\dir_a\note-copy.txt'
Rename-Item '.\dir_a\note-copy.txt' 'note-renamed.txt'
Move-Item '.\dir_a\note-renamed.txt' '.\dir_b\'

Remove-Item '.\empty.txt'
Get-ChildItem -Recurse -Force

Set-Location '..'
```

#### 확인 포인트
- `cli_lab` 아래에 `dir_a`, `dir_b`, `note.txt` 생성 여부 확인
- 최종적으로 `dir_b\note-renamed.txt`가 존재하는지 확인

## 5. Docker 실습 명령어 세트

#### 실행
```powershell
$env:Path += ';C:\Program Files\Docker\Docker\resources\bin'

docker images
docker ps -a
docker logs hello-test
docker rm hello-test
docker stats --no-stream
```

### 9-1. 기본 확인

#### 목적

#### 실행
```bash

```

#### 확인 포인트

---

### 9-2. 권한 변경

#### 목적
- 디렉토리/파일 권한 설정

#### 실행
```bash
chmod 755 dir_a
chmod 644 file_a.txt
```

#### 확인 포인트
- `755` = 소유자 rwx / 그룹 rx / 기타 rx
- `644` = 소유자 rw / 그룹 r / 기타 r

---

### 9-3. PowerShell에서 WSL 권한 실습 실행

#### 목적
- PowerShell에서 한 번에 WSL 권한 실습 수행

#### 실행
```powershell
wsl -d Ubuntu-22.04 -- bash -lc "
rm -rf ~/perm_lab &&
mkdir -p ~/perm_lab/dir_a &&
touch ~/perm_lab/file_a.txt &&
echo 'sample' > ~/perm_lab/file_a.txt &&
echo '--- before ---' &&
ls -ld ~/perm_lab/dir_a &&
ls -l ~/perm_lab/file_a.txt &&
chmod 755 ~/perm_lab/dir_a &&
chmod 644 ~/perm_lab/file_a.txt &&
echo '--- after ---' &&
ls -ld ~/perm_lab/dir_a &&
ls -l ~/perm_lab/file_a.txt
"
```

#### 확인 포인트
- 전/후 권한이 바뀌는지
- 디렉토리와 파일 권한이 각각 올바르게 표시되는지
