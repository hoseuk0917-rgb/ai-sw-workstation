## 상세 내용

### 5-1: 컨테이너 개념과 런타임

컨테이너의 정의와 가상머신과의 차이를 학습합니다. Docker 이미지의 레이어 구조를 이해하고, containerd, CRI-O, Docker Engine 등 3종 컨테이너 런타임을 비교합니다.

### 5-2: 컨테이너 이름 관리

`docker run --name` 옵션으로 컨테이너에 이름을 지정하는 것과 자동 생성 이름의 차이를 실습합니다. 컨테이너 관리 시 명명 규칙의 중요성을 학습합니다.

### 5-3: 포트 매핑

`-p` 옵션을 활용한 호스트 포트와 컨테이너 포트 매핑을 실습합니다. 외부에서 컨테이너 내부 서비스에 접근하는 네트워크 구조를 이해합니다.

### 5-4: Dockerfile 최적화

`.dockerignore` 파일의 역할과 작성법을 학습합니다. Flask 내장 서버 대신 Gunicorn을 프로덕션 환경에서 사용해야 하는 이유를 비교합니다.

### 5-5: Linux vs Windows 컨테이너

Linux 컨테이너와 Windows 컨테이너의 차이를 학습합니다. 각 환경에서의 호환성 문제와 선택 기준을 이해합니다.

### 5-6: 컨테이너 레지스트리

DockerHub를 사용하는 이유와 대안 레지스트리(AWS ECR, Google GCR, Azure ACR 등)를 비교합니다. 이미지 push/pull 워크플로우를 실습합니다.

### 5-7: OS별 실행 차이와 WSL2

Windows, macOS, Linux에서 Docker 컨테이너를 실행할 때의 차이를 학습합니다. Windows에서 WSL2가 필요한 이유와 설정 방법을 다룹니다.
