## 2. PowerShell 기본 명령어 세트

### 2-1. 현재 위치와 목록 확인
```powershell
Get-Location
Get-ChildItem
Get-ChildItem -Force
Get-ChildItem -Recurse -Force
tree /F
```

설명:
- `Get-Location` : 현재 작업 경로 확인
- `Get-ChildItem` : 현재 폴더 목록 확인
- `-Force` : 숨김 항목 포함
- `-Recurse` : 하위 폴더까지 재귀적으로 확인
- `tree /F` : 폴더/파일 트리 구조 확인

### 2-2. 폴더/파일 생성
```powershell
New-Item -ItemType Directory -Path '.\sample_dir'
New-Item -ItemType Directory -Force -Path '.\dir_a','.\dir_b'
New-Item -ItemType File -Path '.\note.txt'
New-Item -ItemType File -Force -Path '.\a.txt','.\b.txt'
```

설명:
- `Directory` : 폴더 생성
- `File` : 파일 생성
- `-Force` : 이미 존재하더라도 오류를 줄이고 생성 시도

### 2-3. 파일 내용 쓰기/읽기
```powershell
'sample line' | Set-Content '.\note.txt'
Get-Content '.\note.txt'
'second line' | Add-Content '.\note.txt'
Get-Content '.\note.txt'
```

설명:
- `Set-Content` : 파일 내용을 새로 작성
- `Add-Content` : 기존 파일 뒤에 이어서 추가
- `Get-Content` : 파일 내용을 읽기

### 2-4. 복사/이동/이름 변경/삭제
```powershell
Copy-Item '.\note.txt' '.\backup_note.txt'
Rename-Item '.\backup_note.txt' 'renamed_note.txt'
Move-Item '.\renamed_note.txt' '.\dir_a\'
Remove-Item '.\a.txt'
Remove-Item '.\temp_dir' -Recurse -Force
```

설명:
- `Copy-Item` : 파일/폴더 복사
- `Rename-Item` : 이름 변경
- `Move-Item` : 다른 위치로 이동
- `Remove-Item` : 삭제
- `-Recurse -Force` : 폴더 전체 강제 삭제

### 2-5. 과제용 PowerShell 기본 실습 세트
```powershell
Set-Location 'D:\코디세이\ai-sw-workstation'

New-Item -ItemType Directory -Force -Path '.\cli_lab' | Out-Null
Set-Location '.\cli_lab'

Get-Location
Get-ChildItem -Force

New-Item -ItemType Directory -Force -Path '.\dir_a','.\dir_b' | Out-Null
New-Item -ItemType File -Force -Path '.\note.txt','.\empty.txt' | Out-Null

'sample line' | Set-Content '.\note.txt'
Get-Content '.\note.txt'

Copy-Item '.\note.txt' '.\dir_a\note-copy.txt'
Rename-Item '.\dir_a\note-copy.txt' 'note-renamed.txt'
Move-Item '.\dir_a\note-renamed.txt' '.\dir_b\'

Remove-Item '.\empty.txt'
Get-ChildItem -Recurse -Force

Set-Location '..'
```

## 3. Git 기본 명령어 세트

### 3-1. 버전 및 상태 확인
```powershell
git --version
git status
git log --oneline --graph --decorate --all
git branch
git branch -a
git remote -v
```

설명:
- `git status` : 변경 상태 확인
- `git log --oneline --graph --decorate --all` : 커밋 이력 요약 확인
- `git branch` : 로컬 브랜치 목록
- `git remote -v` : 원격 저장소 확인

### 3-2. 저장소 초기화 및 기본 설정
```powershell
git init
git config user.name "고호석"
git config user.email "hoseok0917@gmail.com"
git config --list
git branch -M main
```

설명:
- `git init` : Git 저장소 초기화
- `git config` : 사용자 이름/이메일 설정
- `git branch -M main` : 기본 브랜치를 main으로 변경

### 3-3. add / commit / push
```powershell
git add .
git add README.md
git commit -m "docs: update README"
git push
git push -u origin main
```

설명:

- `git add README.md` : 특정 파일만 추가
- `git commit -m "..."` : 커밋 생성
- `git push` : 원격 저장소에 반영

### 3-4. 원격 저장소 연결

## 4. Docker 기본 명령어 세트

### 4-1. Docker 설치/점검
```powershell
docker --version
docker info
where.exe docker
Get-Command docker -ErrorAction SilentlyContinue
```

설명:
- `docker --version` : Docker CLI 버전 확인
- `docker info` : Docker 엔진/환경 상태 확인
- `where.exe docker` : docker 실행 파일 경로 확인

### 4-2. 현재 세션 PATH 추가
```powershell
$env:Path += ';C:\Program Files\Docker\Docker\resources\bin'
```

설명:
- 현재 PowerShell 세션에서 Docker CLI 경로를 PATH에 추가
- 새 PowerShell 창에서는 다시 추가가 필요할 수 있음

### 4-3. 이미지 관련 명령
```powershell
docker images
docker pull hello-world
docker pull ubuntu
docker pull nginx:alpine
docker rmi hello-world
```

설명:
- `docker images` : 로컬 이미지 목록 확인
- `docker pull` : 이미지 다운로드
- `docker rmi` : 이미지 삭제

### 4-4. 컨테이너 관련 명령
```powershell
docker ps
docker ps -a
docker stop 컨테이너명
docker start 컨테이너명
docker rm 컨테이너명
docker rm -f 컨테이너명
```

설명:
- `docker ps` : 현재 실행 중 컨테이너
- `docker ps -a` : 종료된 것 포함 전체 컨테이너
- `docker stop/start` : 중지/시작
- `docker rm` : 삭제

### 4-5. 로그와 상태 확인
```powershell
docker logs 컨테이너명
docker logs -f 컨테이너명
docker stats
docker stats --no-stream
```

설명:
- `docker logs` : 컨테이너 로그 확인
- `docker logs -f` : 로그를 계속 따라가며 보기
- `docker stats` : CPU/메모리 사용 상태 확인

### 4-6. 컨테이너 내부 명령 실행
```powershell
docker exec -it 컨테이너명 bash
docker exec 컨테이너명 bash -lc "명령어"
```

설명:
- `docker exec` : 이미 실행 중인 컨테이너 안에서 명령 실행
- `-it` : 대화형 셸 진입
- `bash -lc "..."` : bash로 한 줄 명령 실행

## 5. Docker 실습 명령어 세트

### 5-1. hello-world 실습
```powershell
$env:Path += ';C:\Program Files\Docker\Docker\resources\bin'
docker run --name hello-test hello-world
docker images
docker ps -a
docker logs hello-test
docker rm hello-test
docker stats --no-stream
```

설명:
- Docker 설치가 정상인지 가장 먼저 확인하는 실습
- hello-world는 실행 후 바로 종료되므로 `docker ps -a`에서 Exited 상태로 보일 수 있음

### 5-2. ubuntu 컨테이너 실습
```powershell
$env:Path += ';C:\Program Files\Docker\Docker\resources\bin'
docker run -dit --name ubuntu-test ubuntu bash
docker exec ubuntu-test bash -lc "ls / && echo hello-from-ubuntu"
docker ps -a
docker logs ubuntu-test
docker rm -f ubuntu-test
```

설명:
- `-d` : 백그라운드 실행
- `-i` : 표준 입력 유지
- `-t` : 터미널 할당
- `docker exec` : 실행 중 컨테이너 내부에서 명령 실행

### 5-3. 직접 bash 진입
```powershell
docker run -it ubuntu bash
```

설명:
- 컨테이너 내부 bash로 즉시 진입하는 가장 단순한 방식
- exit 하면 컨테이너가 종료될 수 있음

## 6. Dockerfile / 웹 서버 / 포트 매핑 예정 명령

### 6-1. nginx 기반 정적 웹 서버 이미지 빌드
```powershell
docker build -t my-web:1.0 .
```

설명:
- 현재 폴더의 Dockerfile을 기준으로 이미지 빌드
- `-t` : 이미지 이름과 태그 지정

### 6-2. 포트 매핑 실행
```powershell
docker run -d -p 8080:80 --name my-web my-web:1.0
```

설명:
- 호스트 8080 포트를 컨테이너 80 포트에 연결
- 브라우저에서 `http://localhost:8080`으로 접속 가능

### 6-3. 포트 매핑 확인
```powershell
docker ps
curl http://localhost:8080
```

설명:
- `docker ps`에서 PORTS 열 확인
- `curl`로 응답 내용 확인 가능

```powershell
docker run -d -p 8080:80 --name my-web-bind -v ${PWD}\app:/usr/share/nginx/html nginx:alpine
```

- 호스트 파일 수정 시 컨테이너 재빌드 없이 내용 반영 가능

### 7-2. 검증 흐름
```powershell
curl http://localhost:8080
notepad .\app\index.html
curl http://localhost:8080
```

설명:
- 수정 전/후 응답을 비교해 변경 반영 여부 확인

## 8. 볼륨 영속성 예정 명령

### 8-1. 볼륨 생성 및 연결
```powershell
docker volume create mydata
docker volume ls
docker volume inspect mydata
docker run -d --name vol-test -v mydata:/data ubuntu sleep infinity
```

### 8-2. 데이터 생성 후 확인
```powershell
docker exec -it vol-test bash -lc "echo hi > /data/hello.txt && cat /data/hello.txt"
docker rm -f vol-test
docker run -d --name vol-test2 -v mydata:/data ubuntu sleep infinity
docker exec -it vol-test2 bash -lc "cat /data/hello.txt"
docker rm -f vol-test2
```

설명:
- 컨테이너는 삭제돼도 볼륨 데이터는 유지됨
- 두 번째 컨테이너에서 같은 파일이 보이면 영속성 검증 성공

## 9. WSL / Linux 권한 명령어 세트

### 9-1. 기본 확인
```bash
pwd
whoami
id
ls -l
ls -ld dir_a
```

### 9-2. 권한 변경
```bash
chmod 755 dir_a
chmod 644 file_a.txt
```

설명:
- `755` = 소유자 rwx / 그룹 rx / 기타 rx
- `644` = 소유자 rw / 그룹 r / 기타 r

### 9-3. PowerShell에서 WSL 권한 실습 실행
```powershell
wsl -d Ubuntu-22.04 -- bash -lc "
rm -rf ~/perm_lab &&
mkdir -p ~/perm_lab/dir_a &&
touch ~/perm_lab/file_a.txt &&
echo 'sample' > ~/perm_lab/file_a.txt &&
echo '--- before ---' &&
ls -ld ~/perm_lab/dir_a &&
ls -l ~/perm_lab/file_a.txt &&
chmod 755 ~/perm_lab/dir_a &&
chmod 644 ~/perm_lab/file_a.txt &&
echo '--- after ---' &&
ls -ld ~/perm_lab/dir_a &&
ls -l ~/perm_lab/file_a.txt
"
```
