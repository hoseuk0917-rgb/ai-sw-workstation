## Codyssey 1주차: AI/SW 개발 워크스테이션 구축 학습 문서

### 1. 미션 개요

이번 1주차 미션은 AI/SW 개발자로서 필수적으로 다루어야 할 **터미널(CLI)**, **Docker(컨테이너)**, **Git/GitHub(버전 관리)**의 기본을 학습하고, 이를 활용하여 **재현 가능한 개발 워크스테이션 환경**을 구축하는 것입니다. "내 컴퓨터에서만 돌아가는" 상황을 방지하고, 누구든 동일한 환경에서 개발, 배포, 디버깅할 수 있는 기반을 마련하는 것이 목표입니다.

### 2. 학습 목표

*   **터미널(CLI)**의 기본 명령어들을 숙지하고 활용하여 파일 및 디렉토리를 관리할 수 있습니다.
*   파일 및 디렉토리의 **권한(Permission)**을 이해하고 `chmod` 명령어를 통해 변경할 수 있습니다.
*   **Docker**의 기본 개념을 이해하고, 설치 및 실행, 컨테이너 관리 (생성, 실행, 종료, 제거, 로그 확인 등)가 가능합니다.
*   **Dockerfile**을 작성하여 자신만의 Docker 이미지를 빌드하고 실행할 수 있습니다.
*   **Docker 포트 매핑**을 통해 호스트와 컨테이너 간의 통신을 설정하고 웹 서비스에 접속할 수 있습니다.
*   **바인드 마운트**와 **Docker 볼륨**을 이해하고, 컨테이너 내외부의 파일/데이터 **영속성** 및 **변경 사항 반영**을 실습할 수 있습니다.
*   **Git**의 기본 설정을 마치고, **GitHub**와 연동하여 코드를 관리할 수 있습니다.
*   **Visual Studio Code (VSCode)**와 GitHub를 연동하여 코드 푸시(Push) 및 풀(Pull) 등의 작업을 수행할 수 있습니다.

### 3. 기능 요구사항

*   터미널을 사용하여 디렉토리 생성, 파일 생성, 복사, 이동, 삭제 등의 기본 작업을 수행해야 합니다.
*   `chmod` 명령어를 사용하여 파일 또는 디렉토리의 권한을 변경하고, 실행 가능하도록 설정해야 합니다.
*   `docker version` 명령어로 Docker가 정상적으로 설치되었는지 확인해야 합니다.
*   `docker run hello-world` 명령어로 Docker 컨테이너 실행 과정을 확인해야 합니다.
*   `docker run -it ubuntu bash` 명령어로 Ubuntu 컨테이너에 접속하고 `pwd`, `ls` 등의 명령어를 실행해야 합니다.
*   `docker attach`와 `docker exec`의 차이를 이해하고 실습을 통해 확인해야 합니다.
*   간단한 웹 서버(예: Nginx)를 위한 `Dockerfile`을 작성하여 커스텀 이미지를 빌드해야 합니다.
*   빌드된 이미지를 사용하여 컨테이너를 실행하고, `docker run -p <host_port>:<container_port>` 명령어를 통해 포트 매핑을 설정해야 합니다.
*   `docker run -v <host_path>:<container_path>` 명령어를 사용하여 바인드 마운트를 설정하고, 호스트의 파일 변경이 컨테이너에 반영되는 것을 확인해야 합니다.
*   `docker run -v <volume_name>:<container_path>` 명령어를 사용하여 Docker 볼륨을 생성하고, 컨테이너가 삭제되어도 데이터가 영속적으로 유지되는 것을 확인해야 합니다.
*   `git config --global user.name` 및 `git config --global user.email` 명령어를 사용하여 Git 전역 설정을 해야 합니다.
*   GitHub 레포지토리를 생성하고, VSCode와 연동하여 코드를 Push/Pull 할 수 있어야 합니다.

### 5. 권장 프로젝트 구조

수강생들의 레포지토리를 참고하여 다음과 같은 구조를 권장합니다.

```
your-github-username/codyssey-m1/
├── README.md               # 프로젝트 설명, 학습 과정, 명령어 로그 등
├── app/                    # Dockerfile 및 관련 파일 (예: index.html)
│   ├── Dockerfile
│   └── index.html          # (선택) 웹 서버 테스트용 파일
├── logs/                   # (선택) 터미널 명령어 실행 로그 또는 스크린샷
│   └── ...
├── screenshots/            # (선택) Docker, VSCode, GitHub 연동 확인 스크린샷
│   └── ...
└── docker-compose.yml      # (선택) Docker Compose 사용 시
```

**각 파일 및 디렉토리 설명:**

*   **`README.md`**: 프로젝트의 목표, 사용 환경, 수행 항목, 명령어 로그, 트러블슈팅 등을 상세히 기록합니다. 학습 내용을 정리하는 핵심 문서입니다.
*   **`app/`**: Dockerfile 및 해당 Dockerfile에서 참조하는 파일들을 모아둡니다.
    *   **`Dockerfile`**: 나만의 Docker 이미지를 만들기 위한 빌드 스크립트입니다.
    *   **`index.html`**: Dockerfile을 통해 웹 서버 컨테이너에 복사되어 웹 브라우저에서 확인할 수 있는 간단한 HTML 파일입니다.
*   **`logs/`**, **`screenshots/`**: 명령어 실행 결과를 기록하거나, 시각적인 증거 자료를 첨부할 때 활용합니다.
*   **`docker-compose.yml`**: 여러 개의 Docker 컨테이너를 묶어서 관리해야 할 경우 사용합니다. (이번 1주차 미션에서는 필수는 아닙니다.)

### 6. 구현 핵심 포인트

1.  **터미널 명령어 숙지:**
    *   `pwd`: 현재 작업 디렉토리의 절대 경로를 확인하는 것은 매우 중요합니다.
    *   `ls -alh`: 파일 및 디렉토리 정보를 자세히 확인하는 데 필수적입니다. 숨김 파일(`-a`)과 사람이 읽기 쉬운 단위(`-h`)로 보는 습관을 들이세요.
    *   `cd ..`: 상위 디렉토리로 이동하는 것은 경로 관리에 유용합니다.
    *   `mkdir`, `touch`: 새로운 작업 공간을 만드는 기본입니다.
    *   `echo "..." > file.txt`: 파일에 내용을 덮어쓰는 방법과 `>>`로 내용을 추가하는 방법(`echo "..." >> file.txt`)을 구분하여 사용하세요.
    *   `cp`, `mv`: 파일 및 디렉토리 복사/이동/이름 변경의 핵심 명령어입니다.
    *   `rm -r`: 디렉토리 및 하위 내용을 삭제할 때 신중하게 사용해야 합니다.

2.  **권한 변경 (`chmod`):**
    *   `chmod 755 <filename>`: 소유자(rwx), 그룹(rx), 타인(rx)에게 실행 권한을 부여합니다. 스크립트 파일을 실행 가능하게 만들 때 주로 사용됩니다.
    *   `rwx`는 각각 숫자 4, 2, 1에 해당하며, 합산하여 권한을 부여합니다. (예: 7 = 4+2+1, 5 = 4+1)

3.  **Dockerfile 작성:**
    *   `FROM <base_image>:<tag>`: 어떤 이미지로부터 시작할지 지정합니다. (예: `nginx:alpine`)
    *   `COPY <host_path> <container_path>`: 호스트의 파일을 컨테이너 내부로 복사합니다. (예: `COPY index.html /usr/share/nginx/html/index.html`)
    *   `EXPOSE <port>`: 컨테이너가 사용할 네트워크 포트를 명시합니다. 실제 포트 매핑은 `docker run`에서 `-p` 옵션으로 설정합니다.

4.  **Docker 컨테이너 실행 및 옵션 활용:**
    *   **`docker run [OPTIONS] IMAGE [COMMAND]`**: 컨테이너를 실행하는 기본 명령어입니다.
    *   **`-d`**: 백그라운드(detached) 모드로 실행합니다.
    *   **`-p <host_port>:<container_port>`**: 호스트의 포트와 컨테이너의 포트를 연결합니다. (예: `-p 8080:80`)
    *   **`-v <host_path>:<container_path>`**: 바인드 마운트. 호스트의 디렉토리를 컨테이너의 디렉토리와 연결하여 호스트에서 파일이 변경되면 컨테이너에 즉시 반영됩니다.
    *   **`-v <volume_name>:<container_path>`**: Docker 볼륨. 컨테이너의 데이터를 영속적으로 저장하기 위해 사용하며, 컨테이너 삭제 후에도 볼륨은 유지됩니다.
    *   **`--name <container_name>`**: 컨테이너에 이름을 지정하여 관리하기 쉽게 합니다.
    *   **`-it`**: 대화형(interactive) 모드로 터미널에 연결합니다. (예: `docker run -it ubuntu bash`)

5.  **Git & GitHub 연동:**
    *   **`git config --global user.name "Your Name"`**: Git에 자신의 이름을 등록합니다.
    *   **`git config --global user.email "your.email@example.com"`**: Git에 자신의 이메일을 등록합니다.
    *   VSCode의 GitHub Extention을 사용하면 GUI 환경에서 편리하게 Git 작업을 수행할 수 있습니다.
    *   `git add .`: 변경된 모든 파일을 스테이징 영역에 추가합니다.
    *   `git commit -m "Commit message"`: 스테이징된 변경 사항을 커밋합니다.
    *   `git push origin <branch_name>`: 로컬 커밋을 GitHub 원격 저장소로 업로드합니다.
    *   `git pull origin <branch_name>`: GitHub 원격 저장소의 최신 변경 사항을 로컬로 가져옵니다.

### 7. 트러블슈팅 & 팁

*   **`echo` 명령어 사용 시 따옴표 문제:**
    *   `echo "Hello World"` 와 같이 큰따옴표(`"`) 안에는 쉘 변수나 특수 문자가 해석될 수 있습니다.
    *   `echo 'Hello World'` 와 같이 작은따옴표(`'`)를 사용하면 문자열 그대로를 전달합니다.
    *   `echo $PATH` 와 같이 환경 변수 값을 그대로 출력하려면 작은따옴표를 사용하거나 `echo \$PATH` 와 같이 이스케이프(`\`) 해야 합니다. (mov-hyun 수강생 참고)

*   **Docker 설치 오류:**
    *   macOS의 경우, Docker Desktop이 정상적으로 설치 및 실행되었는지 확인하세요. OrbStack과 같은 Docker 대체 솔루션을 사용할 수도 있습니다. (codewhite7777 수강생 참고)
    *   Linux의 경우, Docker CE 설치 가이드를 따라 올바르게 설치하고 `docker` 서비스가 실행 중인지 확인하세요.

*   **권한 부족 오류:**
    *   Docker 명령어를 실행할 때 `permission denied` 오류가 발생하면, 현재 사용자가 `docker` 그룹에 속해 있는지 확인하거나 `sudo`를 붙여 실행해야 할 수 있습니다. (Linux 환경에서 주로 발생)

*   **포트 충돌:**
    *   `docker run -p 8080:80`과 같이 포트 매핑 시, 이미 호스트 머신에서 해당 포트(8080)를 사용 중이면 오류가 발생합니다. 다른 포트로 변경하거나 기존 서비스를 중지해야 합니다. (mov-hyun 수강생 참고)
    *   두 번의 포트 매핑 실습은 서로 다른 포트를 사용하여 충돌을 피하는 것이 좋습니다. (예: 8080:80, 8081:80)

*   **VSCode GitHub 연동:**
    *   VSCode의 GitHub Extension을 설치하고 GitHub 계정으로 로그인하세요.
    *   Git Credential Manager가 설치되어 있어 비밀번호를 직접 입력하지 않아도 되도록 설정하는 것이 편리합니다.

*   **바인드 마운트 vs 볼륨:**
    *   **바인드 마운트 (`-v host_path:container_path`)**: 호스트의 특정 디렉토리를 컨테이너와 공유합니다. 개발 중 코드를 수정하고 바로 컨테이너에 반영할 때 유용합니다. 호스트의 파일 시스템에 직접 의존합니다.
    *   **볼륨 (`-v volume_name:container_path`)**: Docker가 관리하는 별도의 저장 공간입니다. 컨테이너가 삭제되어도 데이터가 유지되며, 백업 및 마이그레이션에 유리합니다. 애플리케이션 데이터를 영구적으로 저장할 때 사용합니다.

## Codyssey 2주차 과제: 나만의 퀴즈 게임 학습 문서

### 1. 미션 개요

본 미션은 Python의 기본 문법과 객체지향 프로그래밍 개념을 활용하여, 사용자가 퀴즈를 풀고, 추가하고, 관리할 수 있는 콘솔 기반 퀴즈 게임을 개발하는 것을 목표로 합니다. 프로그램 종료 후에도 퀴즈 데이터와 최고 점수가 유지되도록 JSON 파일을 활용한 데이터 영속성을 구현합니다. 또한, Git을 활용하여 개발 과정을 효율적으로 관리하고 협업 역량을 강화합니다.

### 2. 학습 목표

*   **Python 기초 문법 복습 및 심화:** 변수, 자료형(int, str, list, dict), 조건문(if, elif, else), 반복문(for, while), 함수, 클래스 등의 개념을 실제 프로젝트에 적용하며 이해도를 높입니다.
*   **객체지향 프로그래밍(OOP) 이해 및 활용:** 클래스(Class)와 객체(Object)의 개념을 이해하고, `Quiz`와 `QuizGame`과 같이 데이터를 encapsulating하고 기능을 메서드로 구현하는 방식을 학습합니다. `__init__` 메서드와 `self`의 역할을 명확히 이해합니다.
*   **데이터 영속성 구현:** JSON 형식의 파일 입출력(File I/O)을 통해 프로그램 실행 중 변경된 퀴즈 데이터와 최고 점수를 저장하고, 프로그램 재실행 시 이를 불러오는 기능을 구현합니다.
*   **예외 처리 및 사용자 경험 개선:** 잘못된 사용자 입력(숫자 외 입력, 범위 초과 등)에 대한 예외 처리를 구현하고, `KeyboardInterrupt` 및 `EOFError` 발생 시에도 안전하게 프로그램을 종료하고 데이터를 저장하는 방법을 익힙니다.
*   **Git 기반 프로젝트 관리:** Git의 기본 명령어(`init`, `add`, `commit`, `push`, `pull`, `branch`, `checkout`, `merge`)를 활용하여 코드 변경 이력을 관리하고, 브랜치를 활용한 기능 개발 및 병합 과정을 실습합니다.
*   **프로젝트 구조화 및 코드 가독성 향상:** 논리적으로 관련된 코드들을 함수와 클래스로 분리하고, 적절한 디렉토리 구조를 설계하여 코드의 재사용성과 유지보수성을 높입니다.

### 3. 기능 요구사항

*   **메인 메뉴 제공:** 사용자에게 퀴즈 풀기, 퀴즈 추가, 퀴즈 목록 보기, 최고 점수 확인, 프로그램 종료 기능을 선택할 수 있는 메뉴를 출력합니다.
*   **퀴즈 풀기:**
    *   저장된 퀴즈 중 사용자 요청 개수만큼 랜덤하게 선택하여 출력합니다.
    *   문제와 함께 선택지를 제공하고, 사용자의 답변을 입력받습니다.
    *   정답 여부를 판별하고 점수에 반영합니다.
    *   퀴즈 풀이를 통해 획득한 점수가 최고 점수보다 높을 경우, 최고 점수를 갱신합니다.
    *   (선택 사항) 선택지 순서를 무작위로 섞어 출력합니다.
*   **퀴즈 추가:** 사용자가 직접 문제, 선택지 4개, 정답 번호를 입력하여 새로운 퀴즈를 추가할 수 있습니다.
*   **퀴즈 목록 확인:** 현재 저장된 모든 퀴즈의 문제 내용을 출력합니다.
*   **최고 점수 확인:** 프로그램 실행 동안 기록된 최고 점수를 출력합니다.
*   **데이터 저장 및 불러오기:**
    *   프로그램이 종료될 때 현재 퀴즈 목록과 최고 점수 데이터를 `state.json` 파일에 저장합니다.
    *   프로그램이 시작될 때 `state.json` 파일에서 데이터를 불러옵니다.
    *   `state.json` 파일이 존재하지 않거나 손상된 경우, 기본 퀴즈 데이터와 초기 최고 점수(0점)로 프로그램을 시작합니다.
*   **예외 처리:**
    *   메뉴 선택 시 숫자 외의 입력이나 유효 범위를 벗어난 입력에 대한 오류 메시지를 출력하고 재입력을 유도합니다.
    *   퀴즈 풀이 시 답변 입력에 대한 오류 처리를 합니다.
    *   `KeyboardInterrupt` (Ctrl+C) 또는 `EOFError` (Ctrl+D) 입력 시 안전하게 프로그램을 종료하고 데이터를 저장합니다.

### 6. 구현 핵심 포인트

#### 6.1. `Quiz` 클래스 설계

*   **목적:** 퀴즈의 데이터와 해당 데이터를 다루는 로직을 캡슐화합니다.
*   **주요 속성:**
    *   `question` (str): 퀴즈 질문 내용
    *   `choices` (list[str]): 4개의 선택지
    *   `answer` (int): 정답 선택지의 번호 (1부터 시작)
*   **주요 메서드 (예시):**
    *   `__init__(self, question, choices, answer)`: 객체 초기화
    *   `display(self)`: 문제와 선택지를 보기 좋게 출력 (선택지 순서 섞기 로직 포함 가능)
    *   `is_correct(self, user_answer)`: 사용자의 답변이 정답인지 확인
    *   `to_dict(self)`: JSON 저장을 위해 딕셔너리 형태로 변환
    *   `from_dict(cls, data)`: 딕셔너리에서 `Quiz` 객체를 생성 (클래스 메서드)

#### 6.2. `QuizGame` 클래스 설계

*   **목적:** 게임 전체의 상태와 동작을 관리하는 "게임 운영자" 역할을 합니다.
*   **주요 속성:**
    *   `quizzes` (list[Quiz]): 현재 등록된 모든 `Quiz` 객체의 리스트
    *   `best_score` (int): 최고 점수
    *   `state_path` (str): `state.json` 파일 경로
*   **주요 메서드:**
    *   `__init__(self)`: `quizzes`와 `best_score` 초기화, `state_path` 설정, `load_state()` 호출
    *   `load_state(self)`: `state.json` 파일을 읽어 `quizzes`와 `best_score` 복원. 파일이 없거나 오류 발생 시 기본 데이터로 초기화.
    *   `save_state(self)`: 현재 `quizzes`와 `best_score`를 `state.json` 파일에 저장.
    *   `run(self)`: 메인 메뉴를 반복적으로 출력하고 사용자 입력을 받아 해당 기능을 실행.
    *   `show_menu(self)`: 메뉴 옵션을 출력.
    *   `play_quiz(self)`: 퀴즈 풀이 로직을 담당. 랜덤 문제 선택, 사용자 입력 처리, 점수 계산, 최고 점수 갱신 등을 포함.
    *   `add_quiz(self)`: 새로운 퀴즈를 입력받아 `Quiz` 객체 생성 후 `quizzes` 리스트에 추가.
    *   `show_quiz_list(self)`: `quizzes` 리스트의 모든 퀴즈 정보를 출력.
    *   `show_best_score(self)`: `best_score`를 출력.
    *   `delete_quiz(self, quiz_index)` (선택 사항): 특정 퀴즈를 삭제하는 기능.
    *   `prompt_number(self, message, min_val, max_val)`: 숫자 입력을 받고 유효성을 검증하는 헬퍼 함수.

#### 6.3. 데이터 영속성 (`state.json`)

*   `json` 모듈의 `json.dump()`와 `json.load()`를 사용하여 데이터를 저장하고 불러옵니다.
*   **저장 형식 (예시):**
    ```json
    {
      "quizzes": [
        {
          "question": "파이썬의 창시자는 누구인가요?",
          "choices": ["귀도 반 로섬", "리누스 토르발스", "제임스 고슬링", "브렌던 아이크"],
          "answer": 1
        },
        // ... 다른 퀴즈들
      ],
      "best_score": 15
    }
    ```
*   `load_state()` 메서드에서는 `try-except` 블록을 사용하여 `FileNotFoundError`나 `json.JSONDecodeError` 등을 처리하고, 데이터가 없을 경우 기본 데이터를 생성해야 합니다.

#### 6.4. 사용자 입력 처리 및 예외 처리

*   메뉴 선택, 퀴즈 추가, 답변 입력 등 사용자로부터 값을 받을 때는 항상 유효성 검사를 수행합니다.
*   `try-except` 구문을 사용하여 `ValueError` (예: `int()` 변환 실패) 및 `IndexError` (리스트 범위 초과) 등을 처리합니다.
*   `input()` 함수를 사용할 때, `KeyboardInterrupt` (Ctrl+C) 또는 `EOFError` (Ctrl+D)가 발생할 수 있으므로 `main` 함수나 `QuizGame.run()` 메서드에서 이를 처리하여 안전하게 종료하고 데이터를 저장하도록 합니다.

#### 6.5. Git 활용

*   **초기 설정:** `git init`으로 저장소 생성 후, `.gitignore` 파일에 `.pyc`, `__pycache__` 등 불필요한 파일들을 추가합니다.
*   **커밋 전략:** 기능 단위로 명확하게 커밋 메시지를 작성합니다. (예: "Feat: 메인 메뉴 기능 구현", "Fix: 퀴즈 풀이 시 점수 오류 수정")
*   **브랜치 활용:** 새로운 기능을 개발할 때는 `git checkout -b <feature_branch_name>`으로 새 브랜치를 생성하여 작업하고, 완료되면 `main` 또는 `develop` 브랜치로 `git merge`합니다. `git log --oneline --graph --decorate --all` 명령어로 브랜치 히스토리를 시각적으로 확인하는 연습을 합니다.

### 7. 트러블슈팅 & 팁

*   **`json.dump()` 오류:** `Quiz` 객체를 직접 `json.dump()`로 저장할 수 없습니다. `Quiz` 객체를 `to_dict()` 메서드를 사용하여 딕셔너리 리스트로 변환한 후 저장해야 합니다. 마찬가지로 `json.load()` 후에는 딕셔너리 리스트를 `Quiz` 객체 리스트로 복원하는 과정이 필요합니다.
*   **`self`의 중요성:** 클래스 내부에서 속성에 접근하거나 다른 메서드를 호출할 때는 반드시 `self.`를 붙여야 합니다.
*   **JSON 파일 깨짐:** `state.json` 파일이 의도치 않게 손상되었을 경우, 프로그램 실행 시 오류가 발생합니다. `load_state()`에서 `try-except`를 통해 이를 감지하고 기본값으로 복구하는 로직이 필수적입니다.
*   **랜덤 선택지 순서:** 퀴즈를 풀 때 선택지 순서가 항상 같으면 예측 가능해집니다. `random.shuffle()` 함수를 사용하여 `Quiz` 객체의 `choices` 리스트를 섞고, 이때 정답 번호도 그에 맞게 조정해야 합니다. (이 로직은 `play_quiz` 메서드나 `Quiz` 클래스의 `display` 메서드 안에 포함될 수 있습니다.)
*   **Git 충돌 해결:** 여러 사람이 같은 파일을 수정하고 병합할 때 충돌이 발생할 수 있습니다. 충돌 발생 시 Git의 안내를 따라 수동으로 코드를 수정하고 다시 커밋해야 합니다.
*   **"No module named 'src'" 오류:** `main.py`에서 `from src.quiz_game import QuizGame`과 같이 `src` 디렉토리 내 모듈을 임포트할 때, `src` 디렉토리가 Python 패키지로 인식되지 않으면 오류가 발생합니다. `src/__init__.py` 파일을 비어있는 상태로 생성하여 패키지로 인식되도록 합니다.

# Codyssey 3주차 과제 학습 문서: Mini NPU Simulator

## 1. 미션 개요

이번 3주차 과제는 **Mini NPU Simulator**를 Python 콘솔 애플리케이션으로 구현하는 것입니다. 이를 통해 십자가(Cross)와 X 패턴을 컴퓨터가 숫자 연산을 통해 어떻게 판별하는지, 그리고 NPU의 핵심 연산인 MAC(Multiply-Accumulate)의 원리를 학습하고 직접 구현합니다.

핵심 내용은 다음과 같습니다:
*   2차원 배열을 이용한 패턴 표현
*   필터(Filter)를 이용한 패턴 인식
*   MAC 연산 (Multiply-Accumulate)
*   라벨 정규화 (Label Normalization)
*   부동소수점 비교 정책 (Epsilon-based comparison)
*   시간 복잡도 분석 (`O(N^2)`)
*   실패 케이스 분석 및 트러블슈팅

## 2. 학습 목표

본 과제를 통해 다음과 같은 내용을 스스로 설명하고 이해할 수 있어야 합니다.

1.  **MAC 연산**의 정의와 동작 방식을 설명할 수 있다.
2.  입력 패턴과 필터를 사용하여 **유사도 점수**를 계산하는 원리를 설명할 수 있다.
3.  `data.json` 파일의 **키(Key) 규칙**과 **라벨(Label) 규칙**을 해석할 수 있다.
4.  **라벨 정규화**가 왜 필요한지, 그리고 어떤 방식으로 이루어지는지 설명할 수 있다.
5.  **Epsilon 기반 비교**가 필요한 이유와 그 작동 방식을 설명할 수 있다.
6.  패턴 크기(`N`)가 커질수록 연산량이 왜 `O(N^2)`로 증가하는지 설명할 수 있다.
7.  발생할 수 있는 실패 케이스(데이터, 스키마, 로직, 수치 비교)를 진단하고 해결 방안을 제시할 수 있다.
8.  AI/ML에서 MAC 연산, 필터, 합성곱(Convolution), 행렬곱(Matrix Multiplication)의 관계를 이해한다.

## 3. 기능 요구사항

*   **패턴 표현:** 2차원 배열(리스트의 리스트)을 사용하여 패턴을 표현합니다.
*   **필터:** 기준이 되는 패턴(예: Cross, X)을 2차원 배열로 정의합니다.
*   **MAC 연산:** 입력 패턴과 필터의 같은 위치에 있는 값들을 곱한 후, 그 결과를 모두 더하여 유사도 점수를 계산합니다.
    *   `score = Σ(pattern[r][c] * filter[r][c])`
*   **라벨 정규화:** 다양한 형식의 라벨(예: '+', 'cross', 'X', 'x')을 미리 정의된 표준 라벨('Cross', 'X')로 통일합니다.
*   **부동소수점 비교:** 실제 연산에서는 미세한 부동소수점 오차가 발생할 수 있으므로, 절대적인 값이 아닌 특정 허용 오차(`epsilon`) 내에서의 동점(UNDECIDED) 처리가 필요합니다.
*   **시간 복잡도 분석:** 패턴 크기 `N`에 따른 MAC 연산 시간 복잡도(`O(N^2)`)를 이해하고, 실제 실행 시간을 측정하여 분석합니다.
*   **데이터 처리:** `data.json` 파일에서 패턴 및 필터 데이터를 읽어와 일괄 처리하는 기능을 구현합니다.
*   **두 가지 실행 모드:**
    1.  **사용자 입력 모드:** 3x3 크기의 필터와 패턴을 직접 입력받아 결과를 확인합니다. (예외 처리: 입력 형식 오류 시 재입력 유도)
    2.  **JSON 분석 모드:** `data.json` 파일을 읽어 모든 테스트 케이스를 자동으로 실행하고 결과를 집계합니다. (예외 처리: 스키마 오류, 크기 불일치 시 해당 케이스만 FAIL 처리)
*   **결과 리포트:** JSON 분석 모드 실행 후, 총 테스트 수, 통과 수, 실패 수, 그리고 실패 케이스 목록을 요약하여 출력합니다.
*   **성능 분석:** 각 크기별 MAC 연산에 소요된 평균 시간 및 연산 횟수(`N^2`)를 측정하고 출력합니다.

## 6. 구현 핵심 포인트

### 1. MAC 연산 구현
*   두 개의 2차원 배열(`pattern`, `filter_`)을 입력받아, 같은 인덱스(`i`, `j`)에 있는 요소끼리 곱한 값을 누적하여 최종 점수를 반환하는 함수를 구현합니다.
*   외부 라이브러리 없이 Python의 이중 `for` 반복문을 사용하여 구현합니다.
*   시간 복잡도는 `O(N^2)`이며, `N`은 필터/패턴의 한 변의 길이입니다.

```python
def mac(pattern, filter_):
    n = len(pattern)
    total = 0.0
    for i in range(n):
        for j in range(n):
            total += pattern[i][j] * filter_[i][j]
    return total
```

### 2. 라벨 정규화 (`normalize_label` 함수)
*   `data.json`의 `expected` 값이나 필터 키(`filters`의 키)에 있는 다양한 형태의 라벨을 내부적으로 통일된 표준 라벨(예: 'Cross', 'X')로 변환하는 함수를 만듭니다.
*   예시: `'+', 'cross' -> 'Cross'`, `'x', 'X' -> 'X'`
*   이렇게 정규화하면 비교 로직이 단순해지고, 데이터 소스별 표기 방식 차이로 인한 오류를 방지할 수 있습니다.
*   규칙은 하나의 함수(`normalize_label`)에 집중시켜 관리하는 것이 유지보수에 용이합니다. (수강생 3 참고)

### 3. Epsilon 기반 동점 처리
*   부동소수점 연산의 오차 때문에 `score_a == score_b`와 같은 직접 비교는 불안정합니다.
*   `abs(score_a - score_b) < EPSILON` (여기서 `EPSILON`은 매우 작은 값, 예: `1e-9`)과 같은 조건으로 두 점수가 거의 같은지 판단합니다.
*   이 경우, 판정 결과를 'UNDECIDED' 또는 '판정 불가'로 처리합니다.

### 4. 시간 측정 및 분석
*   MAC 연산 자체의 성능을 측정하기 위해, `time.perf_counter()`를 사용하여 함수 호출 직전과 직후의 시간을 측정하고 그 차이를 계산합니다.
*   I/O 작업(입력, 출력, 파일 읽기) 시간은 측정에서 제외하여 순수한 연산 시간만을 측정합니다.
*   반복 횟수(예: 10회)를 늘려 평균을 구함으로써 일시적인 시스템 부하에 의한 측정 오류를 줄입니다.

### 5. `data.json` 파싱 및 필터 매칭
*   `data.json`의 패턴 키(`size_{N}_{idx}`)에서 패턴의 크기 `N`을 추출합니다.
*   추출된 `N` 값을 이용하여 `filters` 딕셔너리에서 해당 크기에 맞는 'Cross' 또는 'X' 필터를 찾아옵니다.
*   `filters`는 `{크기N: {"Cross": ..., "X": ...}}`와 같은 구조로 미리 준비하면 `filters[n]`으로 `O(1)`에 접근 가능합니다.
*   패턴 키 형식이 맞지 않거나 `N` 파싱에 실패하는 경우, 해당 케이스는 건너뛰고 오류 메시지를 출력하되 프로그램은 계속 실행되도록 예외 처리를 합니다.

### 6. AI/ML 맥락에서의 MAC 연산

*   **MAC vs 행렬곱 vs 합성곱:**
    *   **MAC:** `pattern[i][j] * filter[i][j]`를 모두 더하는 가장 기본적인 연산. 이번 과제의 핵심.
    *   **행렬곱:** 여러 MAC 연산(내적)을 조합하여 결과 행렬을 생성. AI 연산의 중요한 부분.
    *   **합성곱(Convolution):** 필터를 입력 이미지의 여러 영역에 슬라이딩하면서 MAC 연산을 반복하여 특징 맵을 생성. 이미지 처리 AI의 핵심.
*   AI 하드웨어(GPU, NPU)는 이러한 MAC 연산을 매우 효율적으로 처리하도록 설계되었습니다. 이번 과제는 이러한 하드웨어의 근간이 되는 연산을 이해하는 데 도움을 줍니다.

## 7. 트러블슈팅 & 팁

*   **입력 오류 처리 (사용자 입력 모드):**

*   **`data.json` 스키마 오류:**
    *   `json.load()` 시 `JSONDecodeError` 발생 가능.

    *   `filters` 딕셔너리에서 `n`으로 필터를 찾을 때 `KeyError`가 발생할 수 있습니다. (존재하지 않는 크기의 패턴이 있는 경우)
*   **부동소수점 비교 오류:**
    *   `score_a == score_b` 대신 `abs(score_a - score_b) < EPSILON`을 사용합니다. `EPSILON` 값을 너무 작게 잡으면 실제 오차를 무시할 수 있고, 너무 크게 잡으면 다른 값을 동점으로 오인할 수 있으니 주의합니다. (일반적으로 `1e-9` 정도가 적절합니다.)
*   **라벨 정규화 누락:**
    *   `data.json`의 `expected` 값이나 필터 키에서 `+`가 `Cross`로, `x`가 `X`로 제대로 변환되지 않으면 비교에서 실패합니다. 비교 로직 전에 항상 `normalize_label()` 함수를 호출하여 정규화를 수행합니다.
*   **시간 측정 범위:**
    *   `time.perf_counter()` 호출 시점을 MAC 연산 함수 호출 직전과 직후로 정확히 지정해야 I/O 시간을 배제할 수 있습니다.
*   **코드 가독성:**
    *   변수명, 함수명을 명확하게 작성하고, 주석을 활용하여 코드의 의도를 설명하면 이해하기 쉬운 코드를 만들 수 있습니다.
    *   핵심 로직 함수(`mac`, `normalize_label`)를 별도 파일로 분리하는 것이 좋습니다.
