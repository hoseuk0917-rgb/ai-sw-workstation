# 코디세이 AI 올인원 과정: 화성 미션 시리즈 (PBL) 학습 가이드

## 1. 미션 개요

'화성 미션 시리즈'는 코디세이 AI 올인원 과정의 PBL(Problem-Based Learning) 기반 과제로, 수강생이 화성 기지에서 발생할 수 있는 다양한 문제 상황을 Python 프로그래밍을 통해 해결하며 실질적인 문제 해결 능력을 함양하도록 설계되었습니다. 각 미션은 실제 시나리오를 바탕으로 하며, 파일 I/O, 데이터 처리, 알고리즘 구현, GUI 개발 등 다양한 프로그래밍 기술을 요구합니다.

본 학습 가이드는 수강생이 각 미션을 효과적으로 수행하고 학습 목표를 달성할 수 있도록 상세한 지침과 핵심 포인트를 제공합니다.

**미션 구성:**

*   **Mars_1: 화성 기지 로그 분석 (파일 I/O, 텍스트 파싱)**
    *   `mission_computer_main.log` 파일을 읽고 분석하여 화성 기지 사고의 원인을 추정합니다.
    *   로그 데이터를 파싱하고, 시간 순서 및 역순으로 정렬하며, 특정 키워드를 포함하는 로그를 추출하고 JSON 형식으로 저장하는 기능을 구현합니다.
*   **Mars-1_2: 화성 기지 자재 분석 및 돔 설계 (CSV 처리, 데이터 분석, 수학적 모델링)**
    *   화성 기지 자재 목록 CSV 파일을 분석하여 위험 물질을 식별하고, 바이너리 파일로 저장하는 기능을 구현합니다.
    *   돔 구조물의 표면적과 화성에서의 무게를 계산하는 공학적 모델링을 수행합니다.
    *   여러 저장소의 부품 데이터를 통합하고 평균 강도를 분석하여 보수 필요 부품을 식별합니다.
*   **Mars-2/5-1: 시저 암호 해독 (암호학 기초, 문자열 처리)**
    *   시저 암호화된 메시지를 해독하는 알고리즘을 구현하여 암호학의 기본 원리를 이해합니다.
*   **Mars-2/5-2: 공학 계산기 (PyQt5 GUI 프로그래밍)**
    *   PyQt5를 활용하여 사용자 친화적인 공학 계산기 GUI를 개발합니다.
*   **Mars-2/5-4: CCTV 시스템 (멀티미디어, 영상 처리)**
    *   멀티미디어 및 영상 처리 기술을 활용하여 화성 기지 CCTV 시스템의 핵심 기능을 구현합니다. (제공된 자료에는 코드 없음)

각 미션은 독립적으로 수행될 수 있으나, 전체 시리즈를 통해 Python 프로그래밍의 다양한 영역을 깊이 있게 탐구하고 실제 문제에 적용하는 경험을 제공합니다.

## 2. 학습 목표

이 PBL 시리즈를 통해 수강생은 다음과 같은 학습 목표를 달성하게 됩니다.

1.  **Python 기본 문법 및 자료구조 숙달:** 변수, 조건문, 반복문, 함수, 리스트, 딕셔너리 등 Python의 핵심 문법과 자료구조를 실제 문제 해결에 적용하는 능력을 강화합니다.
2.  **파일 I/O 및 텍스트 처리 능력 향상:**
    *   다양한 형식(`.log`, `.csv`, `.json`, `.bin`)의 파일을 읽고 쓰는 방법을 익힙니다.
    *   파일에서 필요한 데이터를 추출하고(파싱), 가공하며, 특정 조건에 따라 필터링하는 기술을 습득합니다.
    *   예외 처리(try-except)를 통해 파일 I/O 과정에서 발생할 수 있는 오류에 안정적으로 대응하는 방법을 배웁니다.
3.  **데이터 분석 및 처리 역량 강화:**
    *   CSV 파일을 파싱하여 데이터를 구조화하고, 특정 기준에 따라 정렬 및 필터링하는 방법을 익힙니다.
    *   `numpy` 라이브러리를 활용하여 대량의 수치 데이터를 효율적으로 처리하고 통계적 분석(평균 계산, 조건부 필터링)을 수행하는 방법을 이해합니다.
    *   데이터를 JSON 및 바이너리 형식으로 직렬화/역직렬화하여 저장하고 로드하는 방법을 학습합니다.
4.  **알고리즘 설계 및 구현 능력 배양:**
    *   시저 암호와 같은 기본적인 암호화/복호화 알고리즘을 이해하고 Python으로 구현합니다.
    *   주어진 문제에 대한 논리적 사고를 통해 효율적인 알고리즘을 설계하고 코드로 구현하는 연습을 합니다.
5.  **객체 지향 프로그래밍(OOP) 기초 이해 (선택적):**
    *   함수 기반의 모듈화된 코드 작성 습관을 기르고, 필요에 따라 클래스를 활용하여 코드의 재사용성과 유지보수성을 높이는 방법을 탐색합니다.
6.  **GUI 프로그래밍 기초 (PyQt5):**
    *   PyQt5 라이브러리를 사용하여 기본적인 데스크톱 애플리케이션의 사용자 인터페이스를 설계하고 구현하는 경험을 합니다.
    *   이벤트 처리 및 위젯(Widget) 활용 방법을 익힙니다.
7.  **문제 해결 및 디버깅 능력 강화:**
    *   주어진 문제 정의를 바탕으로 요구사항을 분석하고, 단계별로 해결 방안을 모색하는 PBL 접근 방식을 체득합니다.
    *   코드 실행 중 발생하는 오류를 식별하고 해결하는 디버깅 능력을 향상시킵니다.
8.  **코드 가독성 및 유지보수성 고려:**
    *   주석 작성, 변수명 명명 규칙 준수, 함수 분리 등을 통해 가독성 높은 코드를 작성하는 습관을 형성합니다.
    *   사용자 입력에 대한 유효성 검사 및 적절한 피드백 제공을 통해 견고한 애플리케이션을 개발합니다.

## 3. 기능 요구사항

각 미션별 핵심 기능 요구사항은 다음과 같습니다.

### Mars_1: 화성 기지 로그 분석

*   **로그 파일 읽기 및 출력:**
    *   `mission_computer_main.log` 파일을 읽어들여 각 라인을 콘솔에 출력해야 합니다.
    *   파일이 존재하지 않거나, 권한 문제, 인코딩 문제 발생 시 적절한 예외 처리를 수행하고 사용자에게 메시지를 전달해야 합니다.
*   **로그 데이터 파싱 및 구조화:**
    *   각 로그 라인(`timestamp,event,message` 형식)을 콤마(`,`) 기준으로 분리하여 `timestamp`, `event`, `message`로 파싱해야 합니다.
    *   파싱된 데이터를 시간 순서대로 출력할 수 있어야 합니다.
*   **로그 데이터 정렬:**
    *   파싱된 로그 데이터를 시간 역순으로 정렬하여 출력할 수 있어야 합니다.
*   **로그 데이터 딕셔너리화 및 JSON 저장:**
    *   로그 데이터를 `timestamp`를 키로 하고 `event`와 `message`를 값으로 하는 딕셔너리 형태로 변환해야 합니다.
    *   변환된 딕셔너리를 `mission_computer_main.json` 파일로 저장해야 합니다. 이때, JSON 파일은 가독성을 위해 `indent=2`로 포맷팅되어야 합니다.
*   **위험 키워드 기반 로그 필터링 및 저장 (Bonus):**
    *   미리 정의된 위험 키워드(`DANGER_KEYWORDS = ["explosion", "unstable", "oxygen", "failure", "malfunction"]`)를 포함하는 로그 라인을 식별해야 합니다.
    *   식별된 위험 로그들을 시간 역순으로 정렬하여 `danger_logs.txt` 파일에 저장해야 합니다.
*   **JSON 파일 내 특정 문자열 검색 (Bonus):**
    *   `mission_computer_main.json` 파일에서 사용자가 입력한 특정 문자열(쿼리)을 포함하는 로그 항목들을 검색하여 시간 역순으로 출력해야 합니다.
    *   검색어가 비어 있거나 JSON 파일이 없거나 파싱 오류가 발생할 경우 적절히 처리해야 합니다.

### Mars-1_2: 화성 기지 자재 분석 및 돔 설계

#### 1. Mars_base_inventory_analyzer.py

*   **CSV 파일 읽기 및 파싱:**
    *   `Mars_Base_Inventory_List.csv` 파일을 읽어들여 각 라인을 콤마(`,`) 기준으로 분리하여 Python 리스트 형태로 변환해야 합니다.
    *   파일이 존재하지 않거나 인코딩 오류 등 발생 시 적절한 예외 처리를 수행해야 합니다.
*   **데이터 정렬:**
    *   읽어들인 자재 목록을 '인화성 지수(Flammability)'를 기준으로 내림차순 정렬해야 합니다.
*   **데이터 필터링:**
    *   인화성 지수가 0.7 이상인 항목들만 필터링하여 별도의 리스트로 생성해야 합니다.
*   **CSV 파일로 저장:**
    *   필터링된 위험 물질 목록을 `Mars_Base_Inventory_Danger.csv` 파일로 저장해야 합니다.
    *   저장 과정에서 발생할 수 있는 권한, 경로, 인코딩, OS 오류 등에 대한 예외 처리를 포함해야 합니다.
*   **바이너리 파일로 저장 및 로드 (Bonus):**
    *   `pickle` 모듈을 사용하여 필터링된 데이터를 `Mars_Base_Inventory_List.bin` 파일로 직렬화하여 저장해야 합니다.
    *   저장된 바이너리 파일을 다시 로드하여 데이터가 올바르게 복원되는지 확인하고 출력해야 합니다.

#### 2. design_dome.py

*   **사용자 입력 처리:**
    *   돔의 바닥 지름(m), 재질(glass, aluminum, carbon_steel 중 택일), 두께(cm, 기본값 1cm)를 사용자로부터 입력받아야 합니다.
    *   입력된 값에 대한 유효성 검사(숫자 형식, 유효한 재질)를 수행하고, 잘못된 입력 시 재입력을 요청해야 합니다.
*   **돔 특성 계산:**
    *   주어진 지름, 재질, 두께를 바탕으로 돔의 표면적(`sphere_area` 함수)과 화성에서의 무게를 계산해야 합니다.
    *   계산 공식:
        *   표면적: `3 * pi * rad^2 + pi * (rad - thick_m)^2` (반구 겉면 + 반구 안쪽면 + 바닥면 고리)
        *   화성 무게: `marsgrav * density * 1000 * 2/3 * pi * (rad^3 - (rad - thick_m)^3)`
        *   `marsgrav = 0.38`, 각 재질별 밀도(`glass=2.4`, `aluminum=2.7`, `carbon_steel=7.85`)
*   **계산 결과 출력:**
    *   계산된 표면적(m²)과 무게(kg)를 소수점 3자리까지 포맷팅하여 사용자에게 출력해야 합니다.
*   **반복 계산 및 종료:**
    *   사용자가 'q'를 입력하기 전까지는 계속해서 돔 특성을 계산할 수 있도록 반복 기능을 제공해야 합니다.

#### 3. Parts_to_work_analyzer.py

*   **여러 CSV 파일 로드 및 병합:**
    *   `mars_base_main_parts-001.csv`, `mars_base_main_parts-002.csv`, `mars_base_main_parts-003.csv` 세 개의 CSV 파일을 `numpy`의 `genfromtxt` 함수를 사용하여 로드해야 합니다.
    *   로드된 `numpy` 배열들을 `np.concatenate`를 사용하여 하나의 배열로 병합해야 합니다.
    *   파일 로드 시 경로 오류, 인코딩 오류 등 예외 처리를 수행해야 합니다.
*   **부품별 평균 강도 계산:**
    *   병합된 데이터에서 각 부품(`parts`)의 고유한 이름을 식별하고, 해당 부품들의 강도(`strength`) 평균을 계산해야 합니다.
    *   `np.unique`, `np.bincount` 등의 `numpy` 함수를 활용하여 효율적으로 계산해야 합니다.
*   **강도 50 미만 부품 식별 및 저장:**
    *   평균 강도가 50 미만인 부품들을 식별해야 합니다.
    *   식별된 부품 이름과 해당 평균 강도를 포함하는 새로운 `numpy` 배열을 생성해야 합니다.
    *   이 배열을 `parts_to_work_on.csv` 파일로 저장해야 합니다. 저장 시 헤더(`parts,strength`)를 포함하고, 소수점 3자리까지 포맷팅해야 합니다.
*   **저장된 CSV 파일 로드 및 전치 (Bonus):**
    *   `parts_to_work_on.csv` 파일을 다시 로드해야 합니다.
    *   로드된 데이터를 전치(transpose)하여 출력하고, `parts_transposed.csv` 파일로 저장해야 합니다.

### Mars-2/5-1: 시저 암호 해독

*   **시저 암호화/복호화 함수 구현:**
    *   주어진 문자열과 이동 키(shift key)를 사용하여 시저 암호화 및 복호화를 수행하는 함수를 구현해야 합니다.

### Mars-2/5-4: CCTV 시스템

*   **영상 스트림 처리 (가정):**

**핵심 개념:**
*   **파일 열기/닫기:** `open()` 함수를 사용하여 파일을 열고, `close()` 또는 `with` 문을 사용하여 안전하게 닫습니다.
*   **읽기/쓰기 모드:** `'r'` (읽기), `'w'` (쓰기, 파일이 없으면 생성, 있으면 덮어쓰기), `'a'` (추가), `'rb'` (바이너리 읽기), `'wb'` (바이너리 쓰기) 등 다양한 모드를 이해합니다.
*   **데이터 형식:** 텍스트 데이터와 바이너리 데이터의 차이점을 이해하고, 각 상황에 맞는 처리 방법을 적용합니다.

**코드 예시 (텍스트 파일 쓰기 및 읽기):**

```python
# 파일 쓰기 예시
def write_mission_log(log_message, filename="mission_log.txt"):
    try:
        with open(filename, 'a', encoding='utf-8') as f:
            f.write(f"{datetime.now()}: {log_message}\n")
        print(f"로그 메시지 저장 완료: {log_message}")
    except IOError as e:
        print(f"파일 쓰기 오류 발생: {e}")

# 파일 읽기 예시
def read_mission_plan(filename="mission_plan.txt"):
    try:
        with open(filename, 'r', encoding='utf-8') as f:
            plan_content = f.read()
        print("\n--- 미션 계획 ---")
        print(plan_content)
        return plan_content
    except FileNotFoundError:
        print(f"오류: '{filename}' 파일을 찾을 수 없습니다.")
        return None
    except IOError as e:
        print(f"파일 읽기 오류 발생: {e}")
        return None

from datetime import datetime
write_mission_log("화성 착륙 성공!")
read_mission_plan("mission_plan.txt") # mission_plan.txt 파일이 미리 생성되어 있어야 함
```

### 6.2. CSV 파싱 (CSV Parsing)

화성 탐사 로봇이 수집하는 센서 데이터(온도, 압력, 습도, 위치 등)나 임무 지시 사항은 종종 CSV(Comma Separated Values) 형식으로 제공됩니다. CSV 파일을 효율적으로 파싱하여 필요한 정보를 추출하고 처리하는 능력은 매우 중요합니다.

**핵심 개념:**
*   **`csv` 모듈:** Python의 내장 `csv` 모듈은 CSV 파일을 쉽게 읽고 쓸 수 있도록 강력한 기능을 제공합니다. `csv.reader`와 `csv.writer`를 주로 사용합니다.
*   **데이터 구조:** 파싱된 CSV 데이터를 리스트의 리스트, 딕셔너리의 리스트 등 적절한 Python 데이터 구조로 변환하여 관리합니다.
*   **오류 처리:** 잘못된 형식의 CSV 파일이나 누락된 데이터에 대한 예외 처리를 고려합니다.

**코드 예시 (CSV 파일 읽기 및 데이터 처리):**

```python
import csv

# CSV 파일 생성 (예시)
def create_sample_sensor_data_csv(filename="sensor_data.csv"):
    data = [
        ['Timestamp', 'Temperature(C)', 'Pressure(Pa)', 'Humidity(%)', 'Latitude', 'Longitude'],
        ['2024-07-20 10:00:00', '25.3', '610.5', '12.1', '1.45', '35.21'],
        ['2024-07-20 10:01:00', '25.1', '610.2', '12.3', '1.46', '35.22'],
        ['2024-07-20 10:02:00', '25.5', '610.8', '12.0', '1.47', '35.23']
    ]
    with open(filename, 'w', newline='', encoding='utf-8') as csvfile:
        csv_writer = csv.writer(csvfile)
        csv_writer.writerows(data)
    print(f"'{filename}' 파일이 생성되었습니다.")

# CSV 파일 읽기 및 파싱 예시
def parse_sensor_data(filename="sensor_data.csv"):
    sensor_readings = []
    try:
        with open(filename, 'r', encoding='utf-8') as csvfile:
            csv_reader = csv.reader(csvfile)
            header = next(csv_reader) # 헤더 읽기
            print(f"CSV 헤더: {header}")
            for row in csv_reader:
                if len(row) == len(header): # 데이터 무결성 확인
                    try:
                        # 데이터 타입 변환 (예시)
                        reading = {
                            'Timestamp': row[0],
                            'Temperature(C)': float(row[1]),
                            'Pressure(Pa)': float(row[2]),
                            'Humidity(%)': float(row[3]),
                            'Latitude': float(row[4]),
                            'Longitude': float(row[5])
                        }
                        sensor_readings.append(reading)
                    except ValueError as ve:
                        print(f"데이터 변환 오류 발생: {row} - {ve}")
                else:
                    print(f"경고: 유효하지 않은 행 발견: {row}")
        print(f"총 {len(sensor_readings)}개의 센서 데이터가 파싱되었습니다.")
        return sensor_readings
    except FileNotFoundError:
        print(f"오류: '{filename}' 파일을 찾을 수 없습니다.")
        return []
    except Exception as e:
        print(f"CSV 파싱 중 오류 발생: {e}")
        return []

create_sample_sensor_data_csv()
parsed_data = parse_sensor_data()
if parsed_data:
    print("\n첫 번째 센서 데이터:", parsed_data[0])
```

### 6.3. 시저 암호 알고리즘 (Caesar Cipher Algorithm)

화성 탐사 중 중요한 통신이나 데이터는 보안을 위해 암호화될 필요가 있습니다. 시저 암호는 가장 기본적인 형태의 암호화 알고리즘으로, 암호화의 원리를 이해하고 구현하는 데 좋은 학습 도구입니다.

**핵심 개념:**
*   **치환 암호:** 각 문자를 일정한 거리만큼 밀어서 다른 문자로 치환하는 방식입니다.
*   **키(Key):** 문자를 얼마나 밀어낼지 결정하는 값입니다.
*   **모듈러 연산:** 알파벳의 끝에 도달했을 때 다시 처음으로 돌아오게 하는 데 사용됩니다 (예: Z 다음은 A).
*   **대소문자 및 특수문자 처리:** 암호화 대상에 따라 대소문자를 구분하거나, 특수문자는 암호화하지 않고 유지하는 등의 처리가 필요합니다.

**코드 예시 (시저 암호화 및 복호화):**

```python
def caesar_cipher(text, shift, mode='encrypt'):
    result = ""
    for char in text:
        if 'a' <= char <= 'z':
            start = ord('a')
            if mode == 'encrypt':
                result += chr((ord(char) - start + shift) % 26 + start)
            else: # decrypt
                result += chr((ord(char) - start - shift + 26) % 26 + start)
        elif 'A' <= char <= 'Z':
            start = ord('A')
            if mode == 'encrypt':
                result += chr((ord(char) - start + shift) % 26 + start)
            else: # decrypt
                result += chr((ord(char) - start - shift + 26) % 26 + start)
        else:
            result += char # 알파벳이 아닌 문자는 그대로 유지
    return result

# 암호화 예시
message = "Mars Mission Control: Landing Confirmed."
shift_key = 3
encrypted_message = caesar_cipher(message, shift_key, mode='encrypt')
print(f"원본 메시지: {message}")
print(f"암호화된 메시지 (Shift {shift_key}): {encrypted_message}")

# 복호화 예시
decrypted_message = caesar_cipher(encrypted_message, shift_key, mode='decrypt')
print(f"복호화된 메시지: {decrypted_message}")
```

### 6.4. PyQt5 GUI (Graphical User Interface)

화성 탐사 로봇의 상태를 모니터링하고, 명령을 내리며, 수집된 데이터를 시각적으로 표현하기 위해서는 사용자 친화적인 GUI가 필요합니다. PyQt5는 Python에서 강력하고 유연한 데스크톱 애플리케이션을 개발할 수 있도록 해주는 라이브러리입니다.

**핵심 개념:**
*   **위젯(Widgets):** 버튼, 레이블, 텍스트 상자, 슬라이더 등 GUI를 구성하는 기본적인 요소들입니다.
*   **레이아웃(Layouts):** 위젯들을 화면에 정렬하고 배치하는 방법 (예: `QVBoxLayout`, `QHBoxLayout`, `QGridLayout`).
*   **시그널과 슬롯(Signals & Slots):** PyQt5의 핵심적인 이벤트 처리 메커니즘으로, 위젯의 특정 이벤트(시그널)가 발생했을 때 특정 함수(슬롯)를 실행하도록 연결합니다.
*   **이벤트 루프:** GUI 애플리케이션이 사용자 입력을 기다리고 처리하는 무한 루프입니다.

**코드 예시 (간단한 PyQt5 GUI):**

```python
import sys
from PyQt5.QtWidgets import QApplication, QWidget, QVBoxLayout, QLabel, QPushButton, QLineEdit, QMessageBox

class MarsControlPanel(QWidget):
    def __init__(self):
        super().__init__()
        self.initUI()

    def initUI(self):
        self.setWindowTitle('화성 미션 제어 패널')
        self.setGeometry(300, 300, 400, 200)

        # 레이아웃 설정
        layout = QVBoxLayout()

        # 위젯 생성
        self.status_label = QLabel('로봇 상태: 대기 중')
        self.command_input = QLineEdit()
        self.command_input.setPlaceholderText('명령을 입력하세요 (예: MOVE FORWARD)')
        self.send_button = QPushButton('명령 전송')

        # 위젯을 레이아웃에 추가
        layout.addWidget(self.status_label)
        layout.addWidget(self.command_input)
        layout.addWidget(self.send_button)

        # 레이아웃을 윈도우에 설정
        self.setLayout(layout)

        # 시그널-슬롯 연결
        self.send_button.clicked.connect(self.send_command)

    def send_command(self):
        command = self.command_input.text()
        if command:
            self.status_label.setText(f'로봇 상태: 명령 "{command}" 실행 중...')
            QMessageBox.information(self, "명령 전송", f"'{command}' 명령이 로봇에 전송되었습니다.")
            self.command_input.clear()
        else:
            QMessageBox.warning(self, "경고", "전송할 명령을 입력해주세요.")

if __name__ == '__main__':
    app = QApplication(sys.argv)
    ex = MarsControlPanel()
    ex.show()
    sys.exit(app.exec_())
```

### 7.1. 일반적인 문제 해결

#### 7.1.1. 인코딩 문제 (UnicodeDecodeError)

**문제:** 파일을 읽거나 쓸 때 `UnicodeDecodeError: 'cp949' codec can't decode byte ...` 또는 유사한 오류가 발생합니다. 이는 주로 한글이 포함된 파일이나 시스템 기본 인코딩과 다른 인코딩으로 저장된 파일을 다룰 때 발생합니다.

**해결책:**
*   **파일 열기 시 `encoding` 명시:** 파일을 열 때 `encoding='utf-8'` 또는 `encoding='euc-kr'`과 같이 명시적으로 인코딩을 지정합니다. UTF-8은 현대적인 표준이며 대부분의 경우에 적합합니다.
    ```python
    # 예시: UTF-8로 파일 읽기
    try:
        with open('data.txt', 'r', encoding='utf-8') as f:
            content = f.read()
    except UnicodeDecodeError:
        print("UTF-8로 디코딩 실패, 다른 인코딩 시도...")
        with open('data.txt', 'r', encoding='euc-kr') as f:
            content = f.read()
    ```
*   **소스 코드 파일 인코딩:** Python 소스 코드 파일 자체도 UTF-8로 저장하는 것이 좋습니다. 대부분의 IDE(VS Code, PyCharm 등)는 기본적으로 UTF-8을 사용하지만, 간혹 문제가 발생하면 파일 인코딩 설정을 확인하세요.
*   **`io` 모듈 사용:** 더 복잡한 인코딩 처리가 필요할 경우 `io` 모듈을 활용할 수 있습니다.

#### 7.1.2. 파일 경로 문제 (FileNotFoundError, PermissionError)

**문제:** `FileNotFoundError: [Errno 2] No such file or directory: 'data.csv'` 또는 `PermissionError: [Errno 13] Permission denied: 'output.txt'`와 같은 오류가 발생합니다.

**해결책:**
*   **절대 경로 vs. 상대 경로:**
    *   **상대 경로:** 현재 스크립트가 실행되는 디렉토리를 기준으로 합니다. `data/mission_plan.json`과 같이 사용합니다.
    *   **절대 경로:** 파일 시스템의 루트부터 시작하는 전체 경로입니다. `C:/Users/User/project/data/mission_plan.json` (Windows) 또는 `/home/user/project/data/mission_plan.json` (Linux/macOS)과 같이 사용합니다.
    *   **팁:** 개발 초기에는 상대 경로를 사용하되, 배포 시에는 `os.path.join`을 사용하여 운영체제 독립적인 경로를 구성하거나, `__file__` 변수를 활용하여 스크립트 파일의 절대 경로를 기준으로 상대 경로를 계산하는 것이 안전합니다.
    ```python
    import os

    # 현재 스크립트 파일의 디렉토리 경로
    current_dir = os.path.dirname(os.path.abspath(__file__))
    data_file_path = os.path.join(current_dir, 'data', 'mission_data.csv')
    ```
*   **경로 존재 여부 확인:** `os.path.exists(path)` 함수를 사용하여 파일이나 디렉토리가 실제로 존재하는지 확인합니다.
*   **권한 문제:** `PermissionError`는 파일이나 디렉토리에 대한 읽기/쓰기 권한이 없을 때 발생합니다. 해당 파일/디렉토리의 권한을 확인하고, 필요한 경우 관리자 권한으로 스크립트를 실행하거나 권한을 변경해야 합니다.

#### 7.1.3. PyQt5 설치 및 실행 문제

**문제:** `ModuleNotFoundError: No module named 'PyQt5'` 또는 PyQt5 애플리케이션 실행 시 오류가 발생합니다.

#### 7.1.4. CSV 파싱 오류 (IndexError, ValueError)

**문제:** CSV 파일을 읽어 데이터를 처리할 때 `IndexError: list index out of range` 또는 `ValueError: could not convert string to float: 'N/A'`와 같은 오류가 발생합니다.

**해결책:**
*   **데이터 형식 확인:** CSV 파일의 각 행이 예상하는 열 개수를 가지고 있는지, 그리고 각 열의 데이터 타입이 올바른지 확인합니다.
    *   **`IndexError`:** 주로 행의 열 개수가 예상보다 적을 때 발생합니다. `len(row)`를 사용하여 행의 길이를 확인하고, 필요한 경우 누락된 데이터를 처리하는 로직을 추가합니다.
    *   **`ValueError`:** 숫자로 변환해야 할 값이 비어있거나, 'N/A', '-' 등 숫자가 아닌 문자열일 때 발생합니다. `try-except` 블록을 사용하여 예외를 처리하고, 기본값 할당, 건너뛰기, 또는 로그 기록 등의 방식으로 처리합니다.
    ```python
    import csv

    data = []
    with open('sensor_data.csv', 'r', encoding='utf-8') as f:
        reader = csv.reader(f)
        header = next(reader) # 헤더 건너뛰기
        for i, row in enumerate(reader):
            if len(row) < 3: # 최소 3개의 열이 필요하다고 가정
                print(f"경고: {i+2}번째 행의 데이터가 불완전합니다: {row}")
                continue
            try:
                timestamp = row[0]
                temperature = float(row[1]) if row[1] else 0.0 # 빈 문자열 처리
                pressure = float(row[2]) if row[2] else 0.0
                data.append({'timestamp': timestamp, 'temperature': temperature, 'pressure': pressure})
            except ValueError as e:
                print(f"경고: {i+2}번째 행에서 데이터 변환 오류 발생 ({e}): {row}")
    ```
